# Brainstorming

## Socratic Protocol

### Phase 1: DEFINE the Problem

- What exactly are we trying to solve?
- Who is affected? How critical is it?
- What constraints exist?

### Phase 2: EXPLORE Options

Generate at least 3 approaches. For each:

- How does it work?
- What are the trade-offs?
- How complex is the implementation?
- Does it fit the existing architecture?

### Phase 3: EVALUATE

| Criteria | Option A | Option B | Option C |
| --- | --- | --- | --- |
| Complexity | ? | ? | ? |
| Maintainability | ? | ? | ? |
| Performance | ? | ? | ? |
| Fits Architecture | ? | ? | ? |
| Time to Implement | ? | ? | ? |

### Phase 4: DECIDE

- Select approach with rationale
- Document in ADR if architectural
- Define implementation plan

## Dynamic Questioning

Ask questions that:

- Challenge assumptions ("What if we didn't need X?")
- Explore boundaries ("What happens at scale?")
- Reveal constraints ("What can't change?")
- Test alternatives ("Could we use Y instead?")

## Anti-Patterns

- Jumping to first solution without alternatives
- Analysis paralysis (set time limit)
- Ignoring constraints (architecture, team size)
- Not documenting the decision
